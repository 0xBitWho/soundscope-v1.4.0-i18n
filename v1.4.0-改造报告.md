# SoundScope v1.4.0 多语言扩展改造报告

> **版本**：v1.4.0
> **改造日期**：2026-06-15
> **开发者**：0xzzyyxx
> **开源协议**：AGPL v3
> **改造范围**：在 v1.3.0 中英文双语基础上，扩展至 9 种语言（含 RTL 支持）

---

## 一、改造目标

在 v1.3.0（简体中文 + 英文）国际化基础上，扩展支持以下 7 种新语言：

| 语言 | 代码 | 资源目录 | 文字方向 |
|------|------|----------|----------|
| 繁体中文 | zh-TW | `values-zh-rTW/` | LTR |
| 法语 | fr | `values-fr/` | LTR |
| 葡萄牙语 | pt | `values-pt/` | LTR |
| 西班牙语 | es | `values-es/` | LTR |
| 阿拉伯语 | ar | `values-ar/` | **RTL** |
| 俄语 | ru | `values-ru/` | LTR |
| 印地语 | hi | `values-hi/` | LTR |

加上原有的简体中文（`values/`）和英文（`values-en/`），**v1.4.0 共支持 9 种语言**。

```
├── values-ru/           # 俄语（新增）

```

**翻译规范**：
- 所有 UI 文本完全本地化翻译

- AI 提示词（`prompt_system_zh`、`prompt_user_zh`、`prompt_system_en`、`prompt_user_en`）在所有语言文件中保持中英文原文不变（这些是发送给 AI 模型的实际提示词，非 UI 文本）
- 语言名称字符串（`lang_chinese`、`lang_english` 等）在各语言文件中保持原生写法

### 2.3 阿拉伯语 RTL 布局支持

`MainActivity.kt` 新增 RTL 布局方向支持：

```kotlin
val isRtl = LocaleManager.isRtl(this)
val layoutDirection = if (isRtl) LayoutDirection.Rtl else LayoutDirection.Ltr
CompositionLocalProvider(LocalLayoutDirection provides layoutDirection) {
    SoundScopeTheme(themeMode = prefs.themeMode) {
        SoundScopeApp(...)
    }
}
```

`LocaleManager.isRtl()` 判断逻辑：
```kotlin
fun isRtl(context: Context): Boolean {
    val locale = getCurrentLocale(context)
    val direction = TextUtils.getLayoutDirectionFromLocale(locale)
    return direction == View.LAYOUT_DIRECTION_RTL
}
```

当用户选择阿拉伯语时，整个应用 UI 自动切换为从右到左布局。

## 五、国际化架构总览

### 5.1 语言管理流程

```
用户在设置页选择语言
        ↓
SettingsScreen 调用 viewModel.updateLanguage(code)
        ↓
UserPreferencesRepository 持久化 languageMode 到 DataStore
        ↓
LocaleManager.applyLanguage(code)
        ↓
AppCompatDelegate.setApplicationLocales(LocaleListCompat.forLanguageTags(code))
        ↓
Activity 重建，应用新语言资源
        ↓
若为阿拉伯语 → MainActivity 检测 RTL → 切换 LayoutDirection.Rtl
```

### 5.2 语言代码映射

| UI 显示 | languageMode | BCP 47 tag | 资源目录 |
|---------|--------------|------------|----------|
| 跟随系统 | `system` | （系统默认） | — |
| 简体中文 | `zh-CN` | `zh-CN` | `values/` |
| 繁体中文 | `zh-TW` | `zh-TW` | `values-zh-rTW/` |
| English | `en` | `en` | `values-en/` |
| Français | `fr` | `fr` | `values-fr/` |
| Português | `pt` | `pt` | `values-pt/` |

### 5.3 可扩展性

当前架构支持未来继续扩展语言，只需：

## 七、已知限制与后续建议

### 7.1 已知限制
- **AI 提示词**：当前 AI 提示词仅中英文两套（按 `LocaleManager.isEnglish()` 切换），其他语言环境下 AI 仍使用英文提示词。如需每种语言独立提示词，需扩展 `AIRepository` 的提示词选择逻辑。
- **图标弃用警告**：`Icons.Filled.Send` 和 `Icons.Filled.ExitToApp` 已弃用，建议后续迁移至 `Icons.AutoMirrored.Filled.*` 版本以更好支持 RTL。
- **Debug 签名**：当前 APK 为 Debug 签名，上架需使用 Release 签名（可复用 v1.2.0 的 keystore）。

### 7.2 后续建议
1. **Release 构建**：使用已有 keystore 生成 Release APK
2. **图标迁移**：将弃用图标替换为 AutoMirrored 版本
3. **AI 提示词扩展**：为 9 种语言各提供独立提示词模板
4. **翻译校对**：建议母语使用者校对 7 种新语言翻译
5. **RTL 细节优化**：阿拉伯语环境下进一步检查图标、动画方向
